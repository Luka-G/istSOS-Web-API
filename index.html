Lightweight Javascript framework for interacting with the sensor observation data managed by IstSOS server implementation

IstSOS Core JavaScript Library

Exposes communication with the IstSOS WA REST interface

IstSOS core JavaScript library was developed using pure JavaScript and Google Closure library, which was particulary used for event handling customization and asynchronous HTTP requests management. IstSOS core JavaScript library covers most of the IstSOS WA REST HTTP requests, which means that developers can easily get the observation data from IstSOS and publish it to their own Web site/page. Using this library, developers can go even further into development and implement IstSOS admin functionalities, like service registering, sensor registering, creating new offerings, adding new units of measures etc...

IstSOS Web Widget Creator

Introduces tools for generating feature-rich widgets based on sensor observation data from IstSOS

IstSOS Web widget creator is an application, that was build using the IstSOS core JavaScript library and IstSOS widget library (particularly designed for widget creation). Tools that are offered to end-users/developers:

  • Map tool
  • Box tool
  • Chart tool (Implementation of VistSOS - IstSOS data visualization framework project. See here...)

Status

IstSOS Web API framework is considered operational from now on, but users/developers should know that, there are some disadvantages that will be worked on in the near future. On the left side below, there is an addition to the Web widget creator usage instructions, which represents restrictive set of rules, when it comes to the usage of more than one widget. On the right side below, there is a list of features that will be fixed, updated, implemented or reimplemented. Some of them are the main reason of the disadvantages, listed on the left side.

List of disadvantages:

If you decide to use AUTO-UPDATE option on MORE THAN ONE widget, make sure that you avoid updating them at the same time. (e.g. set different startup delay, and the same update interval...)

There is no AUTO-UPDATE option for chart widgets, because charts require time interval specified in advance. So avoid using more than one chart widget at the same page. If you really must use it, use JavaScript setTimeout() method as a workaround to avoid using them at the same time.

If you are using map widget or box widget or both, together with the chart widget, keep in mind that chart widget has no startup delay and will update itself at once. So, you should make sure, that map widget and box widget have different startup delay than the chart widget, an also different startup delay between each other.

Features to be implemented/fixed:

EVENT HANDLING

Custom event handling of the istSOS Javascript core library is the main reason of the listed disadvantages. Finding solution for this problem, will be the main goal in the near future.

INSERT OBSERVATIONS

IstSOS Javascript core library doesn't include a method for inserting the observation data.

WEB WIDGET CREATOR DESIGN

During the GSoC, instructions were to focus on functionality, rather than on design. So, Web page containing istSOS Web widget creator application has some design responsivity issues, unorganized HTML usage etc...

BOX WIDGET

Right now, Box widget requires Bootstrap for it's design. Plan is to avoid that and use pure CSS.

MAP WIDGET

Map widget uses OpenLayers 3 for map building. But, when the embedded code is used, map controls like ZoomIn, ZoomOut and other, become invisible, because there is no OL3 css in that code.

  • Week 12

    The state of the project as it was before your GSoC?

    Before my GSoC, IstSOS data couldn't be published on the Web, without hard coding from scratch. Developers would have had to learn about WA REST interface, to implement those requests in order to get the data, to define how they want to publish that data, etc...

    The addition that your project brought to the software?

    With istSOS Web API, developers can use powerfull istSOS core JavaScript library that covers most of the WA REST HTTP requests. Also, developers now have istSOS Web widget creator , at their disposal. This is an application that uses istSOS core Javascript library and offers the set of widget creator tools:

    • Map - developers can easily build map widgets, that show sensor locations, last observation and date&time of the last observation for each sensor
    • Box - developers can easily build boxes, that contain sensor name, date&time of the last observation, and last observation values for each observed property related to selected sensor
    • Chart - integrated framework from the project - VistSOS - Data visualization framework. Developers can easily configure and build several types of interactive charts, that represent observation data in selected time span.
